Pelican for Static Blogging

Background

I'm an opinionated technology fan with various eclectic thoughts to share, so why not add another blog to the universe. I mean who wouldn't want to hear my thoughts on everything from using Python to automate my tasks at work to what type of toilet brush you should be using.

While I'm a fan of technology, more importantly, I'm cheap and want to host this for exactly $0: enter GitHub pages and Pelican. I do a fair bit of Python programming at work and at play, so I liked the sound of a Python based static site generater tool rather than the Ruby based Jekyll. Now don't get me wrong, I don't have anything against Ruby or Jekyll, I just decided to start with Pelican since it is Python based.

Software

I'm starting with Pelican as the blogging engine. I will be writing my content with Markdown. I will be working in a virtualenv setup to generate all my content.

Installing Pelican and Markdown is dead simple.

You will need to create a virtualenv first:

virtualenv pelicanenv

This will create a pelicanenv directory that will house your Python executible and install other tools (e.g. pip) that you can use to install Pelican and Markdown.

You will need to activate your pelicanenv

. pelicanenv/bin/activate

Once you have done this you can install pelican and markdown:

pip install pelican markdown

This will use the pip installation in your pelicanenv and install the latest pelican and markdown to the site-packages directory.

I'll leave it to you to read through the Getting Started guide for the rest of it, but this will get you started.

Other tools

I am using Bitbucket to store the source of the blog and publishing to GitHub. Why you ask? Did I mention that I am cheap? Bitbucket allows me to have private Git repositories for free. GitHub allows me to host my static content for free. I believe Bitbucket will allow me to do the same (see this), but is very undocumented. I may experiment with that, but for now I'll use the better documented GitHub Pages.

I am using the excellent SourceTree for working with my Git repositories.

I am using Sublimet Text as my text editor of choice.

Future

This is obviously the very basics of my new blog, and I will continue to write posts as I go through learning Pelican. Right now I'm sticking with the default theme that ships with it, but I will be experimenting with other themes and write about my experiences.

In the future I may turn on Disqus comments. I will just wait and see how much abuse I am willing to open myself up to.
